IHRP SOP

Single-page checklist for admin tasks and end-to-end payroll runs. Checkboxes persist on this device via localStorage["sop_checks"].

Print-ready • window.print()
Tip: Expand sections, then check items as you go.
1) Upload a new payroll file
/payroll POST /payroll/upload Admin only
[open]
Where to find
Export the AM’s payroll spreadsheet from the source system as an .xlsx file (Excel).
Format
Must be .xlsx (validated by MIME type). Upload limit is 50MB.
What upload does
Parses the spreadsheet, stores the file in storage/app/uploads/payroll, upserts pay period records, replaces consultant entry rows for affected years, and logs an audit entry. It also auto-creates consultant records if a name doesn’t exist yet, and creates/updates raw-name → consultant mappings per AM.
⚠️ Gotchas
  • Wrong AM selected: upload is stored under that AM and replaces entries for the affected years for that AM.
  • Stop name mismatch: if the “stop reading at…” value doesn’t match what’s in the file, parsing can include/exclude unintended rows.
  • Missing bill rates: upload will store revenue = 0.0000 and margin = 0.0000 when a consultant lacks bill_rate.
2) Recompute margins
POST /payroll/recompute-margins Admin only
[open]
Where
From /payroll → open the upload modal (admin) → click Recompute Margins.
When to use
After adding or correcting consultant bill_rate values, especially after fixing multiple consultants or mappings.
What success looks like
Message: “Margins recomputed for N consultant entries.” Dashboard KPIs reflect updated revenue/margin.
⚠️ Gotchas
  • Admin-only: non-admins cannot run this; use an admin account.
  • Does not change AM earnings: am_earnings is treated as immutable source-of-truth from the uploaded Excel file.
  • Bill rate required: entries with missing bill_rate remain at revenue = 0.0000 until the consultant record is updated.
3) Add / edit a user
/admin/users Admin only
[open]
Create user
GET /admin/users/create then submit to create.
Edit user
GET /admin/users/{id}/edit then save changes.
Roles
  • admin: full access (settings, backups, audit log, payroll uploads/recompute, etc.)
  • account_manager: day-to-day views; payroll dashboard scoped to their own data; cannot upload payroll or recompute margins.
⚠️ Gotchas
  • You can’t deactivate yourself: the app blocks self-deactivation from the admin UI.
  • Role controls visibility: payroll uploads and margin recompute are admin-only; AMs won’t see those buttons on /payroll.
4) Update consultant-to-AM mappings
/payroll GET /payroll/api/mappings PUT /payroll/api/mappings
[open]
Where
On /payroll (admin) there is a mappings panel/table for unresolved raw names (loaded via /payroll/api/mappings).
Purpose
Maps raw consultant names found in uploads to an actual consultant record, per AM. Prevents duplicates and fixes mismatches.
What success looks like
The unresolved name disappears from the list after Save (panel refreshes).
⚠️ Gotchas
  • Per-AM mapping: the same raw name can exist under different AMs; map within the correct AM context.
  • Bill rate still required: even with correct mapping, missing consultant bill_rate still yields revenue = 0.0000.
5) Settings (SMTP, logo, backups)
/settings /backups Admin only
[open]
SMTP config
GET /settings and use the SMTP test action (POST /settings/test-smtp) after updates.
Logo upload
From Settings, upload a logo (POST /settings/logo) used in PDFs/emails where applicable.
Backup download
GET /backups to create and download backups (POST /backups then GET /backups/{backup}).
⚠️ Gotchas
  • Admin-only: settings and backups are restricted to admins.
  • SMTP failures: after changing SMTP creds, always run the test action before relying on invoice sends.
1) Pre-run checklist (bill rates)
/consultants Bill rate required
[open]
Goal
Ensure every active consultant that will appear in payroll has a bill_rate set.
Why
Agency revenue is computed as hours × bill_rate. Missing bill rate yields revenue = 0.0000.
⚠️ Gotcha If bill rates are missing, the payroll dashboard will show consultants with revenue = 0.0000 until corrected and margins are recomputed.
2) Export payroll spreadsheet from source system
.xlsx
[open]
What to export
Export the AM payroll spreadsheet for the run as an .xlsx file.
What success looks like
The file opens in Excel and contains the full pay period rows for the run.
⚠️ Gotcha Upload only accepts .xlsx. If you export as another format, convert to .xlsx first.
3) Upload via IHRP
/payroll Upload button (admin)
[open]
Where
GET /payroll → Upload payroll file (admin) → submit to POST /payroll/upload.
What success looks like
“Imported X periods. Years: …” and dashboards update for the selected year/AM.
⚠️ Gotcha If the wrong AM is selected in the upload modal, the import will apply to that AM’s records.
4) Verify dashboard KPIs look correct
/payroll KPIs + pay periods
[open]
What to look for
KPIs (YTD net/gross, taxes, projection), pay period table updates, and charts rendering without errors.
Success looks like
Numbers move in the expected direction compared to the previous run; pay period list includes the new check date(s).
⚠️ Gotcha If something looks off after upload, re-check AM selection and the “stop reading at…” value used during parsing.
5) Check for ⚠️ consultants with revenue = 0
/payroll Meaning: bill_rate missing
[open]
What it means
revenue = 0.0000 indicates missing consultant bill_rate (or zero hours). Revenue requires bill rate.
Success looks like
No active consultants with meaningful hours show revenue = 0 (unless legitimately zero hours).
⚠️ Gotcha Missing bill rate does not break the upload; it silently stores revenue/margin as zero until corrected.
6) If revenue = 0 found: mappings → bill_rate → recompute
/payroll /consultants POST /payroll/recompute-margins
[open]
Fix sequence
Ensure name mapping is correct (if needed), then set consultant bill_rate, then recompute margins for the AM.
Success looks like
After recompute, consultant revenue/margin populate and KPIs update.
⚠️ Gotcha Recompute operates per-AM selection. Make sure the recompute target AM matches where the revenue=0 appeared.
7) Send invoices / run reports
/invoices /reports
[open]
Invoices
Use /invoices to generate/preview/send invoices (and exports as needed).
Reports
Use /reports for monthly and year-end outputs.

Invoices generate without errors, previews look correct, emails send (SMTP ok), and reports download as expected.
⚠️ Gotcha If invoices won’t send, verify SMTP first at /settings using the test action before retrying.
